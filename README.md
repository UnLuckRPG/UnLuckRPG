# 객체 지향 개념 적용 예

## 추상화의 예시

```java
public class Unit {
    private final int maxHp;
    private int hp;
    ...
}

public class Enemy extends Unit
public class Player extends Unit
```

적(`Enemy` )과 플레이어(`Player` )는 공통적으로 최대 hp와 maxHp를 가지고 있다.

따라서 부모 클래스 유닛(`Unit` )의 속성으로 정의한다.

## 상속의 예시

```java
public abstract class Item {
    public String getName() {
        return name;
    }
}

public abstract class Potion extends Item
public class extends NormalPotion
```

`Item` 의 구현체 `NormalPotion` 에서도 부모의 `getItem()` 을 간편하게 이용할 수 있다.

## 캡슐화의 예시

```java
 public class Game
    private static boolean isGameOver;
    public static void setGameOVer() {
        isGameOver = true;
    }
    ...
 }
```

`isGameOver` 속성을 보호하고 외부로부터 필요한 메소드 `setGameOver()` 를 노출한다.

따라서 외부 클래스에서는 `isGameOver` 를 false로 바꿀 수 없다.

## 다형성의 예시

```java
// Player.java
public boolean addItem(Item item) {
		...
}

// Shop.java
player.addItem(new NormalPotion()); // NomalPotion은 Item의 자식 클래스

```

`addItem` 의 시그니쳐가 `Item` 으로 정의되어있기 때문에 물약인지 주사위인지 신경쓰지 않고 아이템을 추가할 수 있다.

# 게임 흐름

1. 게임이 시작되면 플레이어가 생성된다.
2. 5개의 지역이 랜덤으로 생성된다.
    - 각 지역은 여관, 상점, 던전 중 하나이다.
3. 플레이어는 생성된 각 지역(스테이지)를 순서대로 통과해야 한다.
    - 플레이어의 hp가 0이 되었을때 플레이어는 쓰러진다.
    - 쓰러진 플레이어는 스테이지를 통과할 수 없다.
    - 스테이지를 통과할 수 없으면 플레이어의 패배로 게임이 종료된다.
4. 플레이어가 모든 지역을 성공적으로 통과했을 시 플레이어의 승리로 게임이 종료된다.

# 요구사항

## 게임(Game)

- 게임을 시작하면 5개의 지역(Region)을 랜덤으로 생성한다.
- 랜덤으로 생성된 지역을 차례대로 입장하고 빠져나온다.
- 5개의 지역을 모두 통과했다면 플레이어는 게임에서 승리한다.

## 여관(Inn)

- 여관에 들어오면 50% 확률로 체력을 전부 회복할 수 있다.
    - 화면에 표시되는 메시지 : 플레이어는 여관에서 체력을 전부 회복하였습니다.
- 나머지 50%의 확률로 아무 일도 일어나지 않는다.
    - 화면에 표시되는 메시지 : 여관이 꽉 차서 휴식할 수 없습니다.
- 이후 여관을 빠져나온다.

## 상점(Shop)

- 상점에 들어오면 선택지가 표시된다.
    - 아이템 구입하기
    - 상인을 협박하기
- ‘아이템 구입하기’를 선택 시 플레이어는 50G를 지불하고 랜덤으로 주사위나 물약을 하나 받는다.
    - 플레이어의 소지금이 부족할 시 ‘소지금이 부족합니다’를 출력하고 선택지가 다시 표시한다.
- ‘상인을 협박하기’를 선택 시
    - 50% 확률로 소지금을 지불하지 않고 주사위나 물약을 하나 받는다.
    - 50% 확률로 플레이어의 현재 hp의 50%를 깎는다.
- ‘아이템 구입하기’나 ‘상인을 협박하기’ 중 하나를 선택했으면 상점을 빠져나온다.
- 주사위를 받았을 시 원래 가지고 있던 주사위를 버린다.

## 던전(Dungeon)

- 던전에 입장 시 ‘쉬운 적’, ‘보통 적’, ‘어려운 적’ 중 하나가 등장한다.
- 50%의 확률로 플레이어가 먼저 공격할 기회를 갖는다.
- 50%의 확률로 적이 먼저 공격할 기회를 갖는다.
- 각 유닛은 자신의 차례에 주사위를 굴린다.
    - 나온 주사위의 눈 수만큼 적의 현재 Hp를 감소시킨다.
    - 유닛의 hp가 0이 되면 전투가 종료된다.
        - 플레이어가 패배하면 게임을 종료한다.
        - 적이 패배하면 던전을 탈출한다.

## 유닛(Unit)

- 유닛은 던전에서 공격할 수 있다.
- hp가 0이 되면 패배한다.

## 플레이어(Player)

- 플레이어는 아이템들을 소지할 수 있다.
- 플레이어의 hp가 0이 되면 쓰러지며 게임에서 패배한다.

# 아이템(Item)

- 아이템은 이름이 존재한다.

## 주사위(Dice)

- 주사위는 최소 눈과 최대 눈이 있다.
- 이름과 성능
    - 일반 주사위 : 최소 눈 1, 최대 눈 6
    - 안정적인 주사위 : 최소 눈 3, 최대 눈 4
    - 극단적인 주사위 : 최소 눈 1, 최대 눈 10
    - 슈퍼 주사위 : 최소 눈 5, 최대 눈 10
- 주사위는 소모되지 않는다.

## 물약(Potion)

- 물약을 마시면 체력이 전부 회복된다.
- 물약은 소모품이다.
